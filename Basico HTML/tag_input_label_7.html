A tag input/ no HTML é usada para coletar dados do usuário, como texto, números, e-mails, etc.
Já a tag label/ é utilizada para criar rótulos que identificam e descrevem os campos de entrada, tornando a interface mais clara e acessível.

label serve para acessibilidade e ao clicar em nome que nosso label ele irar da foco ao input do nome.
As labels podem ser associadas aos inputs de forma explícita (usando o atributo for) ou implícita (colocando o input dentro do label).

tag input

  • type: define o tipo de entrada que o campo aceitará. Exemplos: `text`, `password`, `email`, `number`, `date`, etc.
  • id: identifica exclusivamente o elemento na página. Usado com a tag `
  • name: nome do campo. Usado na submissão do formulário para enviar os dados ao servidor.
  • valor: valor padrão do campo. No envio do formulário, esse valor será enviado.
  • placeholder: texto exibido dentro do campo como dica temporária (desaparece ao digitar).
  • required: torna o preenchimento obrigatório para o envio do formulário.
  • readonly: o campo pode ser lido, mas não pode ser alterado pelo usuário.
  • disabled: desativa o campo. Ele não pode ser editado nem enviado.
  • maxlength: define o número máximo de caracteres permitidos no campo.
  • minlength: define o número mínimo de caracteres obrigatórios.
  • min: valor mínimo permitido (para `number`, `date`, `range`, etc.).
  • max: valor máximo permitido (para `number`, `date`, `range`, etc.).
  • passo: definir o intervalo de valores válidos. Exemplo: `step="5"` permite 0, 5, 10...
  • size: define a largura do campo de texto em número de caracteres visíveis (apenas visual).
  • pattern: expressão regular que define um padrão de texto aceito (validação).
  • checked: usado em `checkbox` e `radio` para marcar o campo como selecionado por padrão.
  • multiple: permite vários arquivos seletivos, como arquivos (`file`) ou múltiplos e-mails.
  • autofocus: faz o campo receber foco automaticamente ao carregar a página.
  • autocomplete: define se o navegador deve sugerir valores com base em preenchimentos anteriores (`on` ou `off`).
  • form: associa o campo de entrada a um `
    ` específico usando o `id` do formulário.
  • list: vincula o campo a um elemento `` para fornecer sugestões de preenchimento.
  • accept: define quais tipos de arquivos podem ser enviados (usado com `type="file"`). Ex: `.jpg`, `image/*`, etc.
  • src: usado apenas com `type="image"`. Defina o caminho da imagem que será usada como botão de envio.
  • alt: texto alternativo alterna se a imagem não for compartilhada, também importante para acessibilidade.
